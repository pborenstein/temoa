# Documentation Archive

Historical documentation: completed plans, superseded docs, and the v1 era
(UI-centric search tool with gleanings, web UI, and graph — rebuilt as a pure
search engine in v2.0, 2026-05-30).

## Directory Structure

### `chronicles-v1/`
Session-by-session chronicles from the v1 era (2025-11 – 2026-05): phases 0–3.5,
production hardening, gleaning work, and the search harness UI. The live
chronicle for the v2 pure-search era is `docs/chronicles/v2-pure-search.md`.

### `original-planning/`
Initial project vision and planning documents (2025-11).

### `phase-2.5-completed/`
Phase 2.5 implementation plans: compact view, management page, gleaning status,
search quality (2025-11).

### `reviews/`
End-of-Phase-2.5 reviews (architecture, search, UI), consolidated into
`PHASE-3-READY.md`.

### `QoL-improvements/`
Quality-of-life UI improvements plan and before-screenshots (2025-12 – 2026-01).

### `backburner/`
Deferred plans (Phase 4: Vault-First LLM).

### Loose files
Completed or superseded plans (incremental indexing, multi-model, gleaning
normalization, UI cleanup, GitHub layout), assessments, and research notes.
`gleanings-history.md` chronicles the gleaning system, which now lives in
pixquitl. `qmd-pipeline-improvements.md` still holds actionable ideas
(position-aware blending, heading-aware chunking).

## When to Archive Documents

Archive when a plan is implemented, superseded, or consolidated. Keep in
`docs/` only living reference material (ARCHITECTURE.md, SEARCH-MECHANISMS.md,
DEPLOYMENT.md) and active tracking (CONTEXT.md, IMPLEMENTATION.md,
DECISIONS.md, chronicles/).
